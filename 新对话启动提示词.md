## 关键技术细节

### 数据流

```
STORE 对象（唯一数据源）
  ├── renderCreatorTable() → 星图达人表格
  ├── renderContentTable() → 内容表现表格
  ├── renderNonStarTable() → 非星图达人表格
  ├── updateKPIs() → 读 #tbl-creator DOM → 更新星图相关 KPI
  └── syncAllDisplayValues() → 读 STORE → 更新种草通/声量宝/星助推/CEO 文本
```

### 关键 JS 函数

- `syncAllDisplayValues()` — **核心**：统一同步所有硬编码 HTML 值
- `updateKPIs()` — 基于 tbl-creator 筛选后的 DOM 计算星图 KPI
- `updateCardKV(pageId, label, value, sub)` — 按 label 文本定位并更新 KPI 卡片
- `updateDetailRow(pageId, label, value)` — 更新详细表的某一行

### 已知坑点

1. **HTML div 嵌套不匹配** — 最常见 bug，导致页面内容不显示。每次编辑 HTML 后必须运行 div 平衡检查。
2. **数据更新三路径** — 改 STORE 后必须：①调 `syncAllDisplayValues()` ②检查品牌广告/原生加热页的硬编码 KPI ③检查跨页引用（如电商页的种草通 A3 数字）
3. **星智投 CPE** — 必须用投放互动量（873,584）做分母，不能用总互动（2,989,498）
4. **非星图达人筛选** — 仅包含水下 KOC/水下 KOL/素人 + 胡连馨儿/王楚然粉丝/王楚然工作室，其他艺人/KOL 已归入星图
5. **种草通数据来源** — 文件列索引：spend@[5], imp@[6], play@[7], like@[8], comment@[9], share@[10], A3@[11]
6. **声量宝数据来源** — 文件列索引：spend@[10], play@[14], like@[16], comment@[17], share@[18], search@[23]
7. **星助推数据来源** — 文件列索引：spend@[4], play@[8], like@[9], comment@[10], share@[11], 星智投play@[17], 星智投like@[18], 星智投comment@[19], 星智投share@[20]
8. **5A CSV** — 必须用 `encoding='utf-8-sig'` 打开

  2. 更新 CEO 成本计算
  3. 更新静态 HTML 值（KPI 卡片 + 详细表 + 跨页引用）
