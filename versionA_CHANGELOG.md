## 版本概述

A 版本是一个基于真实数据的抖音达人种草营销数据看板原型，支持三大投放目的 × 自然/付费流量拆分 × 多平台产品矩阵。前端纯静态 HTML，支持三层密码权限，所有数据表支持排序/筛选并联动 KPI 动态更新。

---

## 开发历程

### Phase 1: 架构设计

**产出物**: `/Downloads/抖音达人种草营销数据看板方案.md`

- 数据源全景图: 星图 / 千川 / 巨量引擎 / 罗盘 / 云图 / 第三方
- 投放目的三分法: 曝光型 / 种草型 / 转化型
- 自然 vs 付费流量拆分矩阵
- 完整产品归属矩阵（巨量引擎 / 千川 / 原生内容加热）
- 8 页面架构（后精简为 7 页面）
- 指标体系：各目的的完整 KPI 树

### Phase 2: 数据库设计

**产出物**: `/Downloads/dashboard_ddl.sql`

- 19 张表：维度表 8 张 + 事实表 4 张 + 缝合表 1 张 + 汇总表 1 张 + 管理表 5 张
- 核心设计：ID-Mapping 三层架构（达人/内容/商品）
- 千川×星图缝合宽表 DDL
- ETL 示例 SQL

### Phase 3: HTML 原型迭代

| 版本 | 关键更新 |
|---|---|
| **v1** | 初始原型：8 页面 + 密码登录 + 指标显隐 + 数据上传入口 |
| **v2** | 数据清理：剔除所有虚构数据，仅保留 rawdata 真实值，修复无效按钮 |
| **v3** | 5A 数据补充：接入 A1/A2/A3 完整数据（7/1→7/15），5A 漏斗可视化 |
| **v4** | 排序筛选 + 联动 KPI：表头升/降序，搜索+层级筛选，筛选后 KPI 自动重算 |
| **v4.1** | 产品层级标签：品牌广告 + 原生加热页添加产品结构标签 |
| **v4.2** | 产品独立分卡：每个产品独立卡片（KPI行+详细表），无数据产品灰色占位 |
