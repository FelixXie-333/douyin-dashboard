## 一、数据源全景图

### 1.1 平台数据角色

| 数据源 | 核心数据 | 种草营销中的角色 |
|---|---|---|
| **巨量星图 (Xingtu)** | 达人报价、粉丝画像、历史商单、自然播放/互动 | 达人筛选 + 内容自然表现 |
| **巨量千川 (Qianchuan)** | 付费投放消耗、展现、转化、ROI | 电商付费转化 |
| **巨量引擎 (Ocean Engine)** | 品牌广告(品牌专区/TopView/FeedsLive)、种草通、AD竞价信息流 | 品牌曝光放大 + 种草内容加热 |
| **抖音原生** | DOU+ / 声量宝 / 星智投 | 原生内容加热 + 搜索撬动 |
| **电商罗盘 (Luopan)** | 商品GMV、搜索成交、内容成交、流量来源 | 转化归因终点 |
| **巨量云图 (Yuntu)** | O-5A人群资产、品牌力指数、搜索洞察 | 品牌资产长期度量 |
| **第三方** (蝉妈妈/飞瓜) | 竞品达人策略、行业benchmark | 竞品对标 |

### 1.2 种草核心链路

```
星图选达人 → 自然内容分发 → 千川加热放大 → 罗盘验证转化 → 云图沉淀人群资产
     │              │              │              │              │
  达人资产看板    内容表现看板    投放分析专区    电商转化看板    品牌资产看板
```

## 二、投放目的三分法

### 2.1 三大目的定义

```
曝光目标 (Exposure):  最大化有效触达 × 控制触达成本 × 验证触达质量
种草目标 (Seeding):   激发深度互动 × 沉淀A3人群 × 控制种草成本
转化目标 (Conversion): GMV最大化 × ROI可控 × 区分投放ROI和全成本ROI
```

### 2.2 全局切换影响范围

```
[ 曝光型 │ 种草型 │ 转化型 ] ← 全局目的切换器

✓ 影响:
  ├── 达人矩阵排序逻辑 (CPM / CPE / ROI)
  ├── 内容排行榜头号指标 (播放量 / 互动率 / GPM)
  ├── 投流优化目标 (CPM / CPE / ROI)
  ├── 素材评估阈值 (衰减标准不同)
  ├── 达人推荐算法权重
  └── 预警规则 (不同目的容忍度不同)

✗ 不影响: 电商转化看板、人群资产原始数据 (仅解读视角变化)
```

### 2.3 各目的完整指标体系

#### 曝光目标

| 类别 | 指标 | 计算方式 |
|---|---|---|
| **传播效率** | 展现量 Impression | 内容被展示次数 |
| | 播放量 VV | 视频播放次数 |
| | 有效播放量 Valid Views | 播放≥5s次数 |
| | 5秒完播率 | 有效播放/展现 |
| | 整体完播率 | 完整看完/播放量 |
| | 平均观看时长 | 总观看时长/播放量 |
| | 触达UV | 去重观看人数 |
| | 触达频次 Frequency | 展现量/触达UV |
| | 非粉触达占比 | 非粉UV/总触达UV |
| | 粉丝增长量/率 | — |
| **成本效率** | **CPM** (千次展现成本) | 投放消耗/展现量×1000 |
| | **CPVV** (有效观看成本) | 投放消耗/有效播放量 |
| | CPV (单次播放成本) | 投放消耗/播放量 |
| | CPF (单粉成本) | 投放消耗/新增粉丝 |
| | CPU (单UV成本) | 投放消耗/触达UV |
| **人群资产** | 新增A1 | 投后A1 - 投前A1 |
| | CPA1 | 投放消耗/新增A1 |
| | A2流转率 | 新增A2/触达UV |
| | 品牌搜索指数变化 | 投后搜索指数 - 投前 |

#### 种草目标

| 类别 | 指标 | 计算方式 |
|---|---|---|
| **互动深度** | 互动量 Engagement | 点赞+评论+分享+收藏 |
| | **互动率** Engagement Rate | 互动量/播放量 |
| | 点赞率 / 评论率 / 分享率 / **收藏率** | 各自/播放量 |
| | 评论正向率 | 正向评论/总评论 |
| | 长播率(>30s) | 观看>30s/播放量 |
| **种草行为** | 搜索引导UV | 观看后主动搜索品牌人数 |
| | 搜索引导率 | 搜索引导UV/播放量 |
| | 购物车点击量/率 | — |
| | 话题参与UV | — |
| | UGC二创数量 | 用户模仿/二次创作数量 |
| **人群资产** | **新增A3** | 投后A3 - 投前A3 |
| | **CPA3** | 投放消耗/新增A3 |
| | A2→A3流转率 | 新增A3/投前A2 |
| | A3→A4流转率 | 新增A4/投前A3 |
| | A3自然占比 | 自然新增A3/总新增A3 |
| **成本效率** | **CPE** (单次互动成本) | 投放消耗/互动量 |
| | CPE-深度 | 投放消耗/(评论+分享+收藏) |
| | CPC-搜索 | 投放消耗/搜索引导UV |

#### 转化目标

| 类别 | 指标 | 计算方式 |
|---|---|---|
| **成交规模** | **GMV** | 支付金额汇总 |
| | 短视频GMV / 直播GMV / 商品卡GMV | 按来源拆分 |
| | 订单量 / 客单价 AOV | — |
| | **GPM** (千次展现成交额) | GMV/展现量×1000 |
| | UV价值 | GMV/触达UV |
| **转化漏斗** | 购物车点击率 → 商品点击率 → 加购率 → 下单率 → 支付转化率(CVR) | — |
| **ROI** | **投放ROI** (仅投放cost) | GMV/投放消耗 |
| | **综合ROI** (含达人佣金+固费) | GMV/(投放消耗+达人佣金+固费) |
| | 自然ROI | 自然GMV/达人总成本 |
| **成本** | CPA (投放口径) | 投放消耗/成交订单数 |
| | 综合CPA (全口径) | (投放消耗+达人费用)/成交订单数 |
| **质量** | 退货率 / 差评率 / 新客成交占比 / 复购率 | — |

## 三、自然 vs 付费流量拆分

### 3.1 核心交叉矩阵

```
                   曝光目标                种草目标                转化目标
              ┌──────┬──────┐      ┌──────┬──────┐      ┌──────┬──────┐
              │ 自然 │ 付费 │      │ 自然 │ 付费 │      │ 自然 │ 付费 │
──────────────┼──────┼──────┤──────┼──────┼──────┤──────┼──────┼──────┤
核心看什么     │自然播放│付费CPM│      │自然互动率│付费CPE│      │自然GPM│付费ROI│
──────────────┼──────┼──────┤──────┼──────┼──────┤──────┼──────┼──────┤
关键追问       │达人自带│投放放大│      │内容真实│加热是否│      │达人自然│付费能否│
              │流量够大│效率    │      │种草力够│加速种草│      │带货力够│放量盈利│
──────────────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
```

### 3.2 关键衍生指标

| 指标 | 计算方式 | 说明 |
|---|---|---|
| 投放杠杆倍数 | (自然+付费播放量)/自然播放量 | >3x=投放友好型 |
| 自然流量贡献率 | 自然播放/总播放×100% | 内容真实力 |
| 加热健康度 | 付费互动率/自然互动率 | >0.7=优质, <0.4=人群不匹配 |
| 付费协同效应 | (投流后自然播放-投流前自然播放)/付费播放 | 正值=反哺, 负值=蚕食 |

## 七、各页面指标分级与显隐规则

### 7.1 指标分级体系

```
★★★ 核心指标  = 始终展示，不可隐藏 (灰锁 🔒)
★★  重要指标  = 默认展示，可隐藏 (☑)

```

### 7.2 各页面详细分级

#### 1. CEO驾驶舱

```
🔒 始终展示 (4):
```

## 八、数据上传中心

### 8.1 上传入口

全局导航栏右侧固定 `[📤 数据上传]` 按钮。

### 8.2 支持的数据源

| 数据源 | 文件类型 | 识别关键词 |
|---|---|---|
| 星图-任务投放数据 | .xlsx/.csv | 包含 "星图任务ID" + "视频ID" |
| 千川-投放报表 | .xlsx/.csv | 包含 "广告计划ID" 或 "campaign_id" |
| 巨量引擎-品牌广告 | .xlsx/.csv | 包含 "品牌专区" 或 "种草通" 列 |
| 巨量引擎-竞价信息流 | .xlsx/.csv | 包含 "ad_id" 或 "创意ID" |
| 罗盘-成交数据 | .xlsx/.csv | 包含 "spu_id" 或 "支付GMV" |
| 云图-人群数据 | .xlsx/.csv | 包含 "5A" 或 "人群包" |
| DOU+-加热数据 | .xlsx/.csv/.手动 | 包含 "DOU+消耗" |

### 8.3 上传处理流程

```
文件上传
  → ① 格式校验 (检查列名匹配度)
  → ② 字段映射预览 (用户确认列名→系统字段)
  → ③ 时间特征标注 (累计/按日/周期快照)
  → ④ 数据去重&合并 (按主键)
  → ⑤ 数据校验&冲突仲裁 (与已有数据对比)
  → ⑥ 入库
```

### 8.4 冲突仲裁策略

| 冲突场景 | 默认处理 | 可选处理 |
|---|---|---|
| 同一视频ID，新数据播放量更大 | 替换为新值 (累计数据更新) | 保留旧值 / 同时保留为快照版本 |
| GMV多源冲突 (千川归因 vs 罗盘实际) | 以罗盘为默认源 | 手动选择千川归因 |
| 全域推广标记冲突 | 全域优先 (不可拆分) | — |

## 九、时间筛选机制

### 9.1 各平台时间粒度差异

```
平台          数据粒度        时间筛选含义                 筛选控件
─────────────────────────────────────────────────────────────
星图          累计快照         筛选 = "截止某日期的累计值"    日期选择器(单日)
              无按日数据       如需按日→需两次快照相减        + 增量模式开关

千川          按日数据         筛选 = "日期范围内每日之和"    日期范围选择器
              天然支持         单日/多日/周/月均可

巨量引擎      按日数据         同千川                         日期范围选择器

罗盘          按日数据         按支付日期筛选                 日期范围选择器

云图          周期快照         筛选 = 选择最近一次快照        周选择器
              T+1周更新        无按日

DOU+          手工填报         按投放日期筛选                 日期范围选择器
              按内容维度

星智投        星图累计快照     同星图逻辑                     单日选择器
─────────────────────────────────────────────────────────────
```

### 9.2 星图累计→增量转换

```
快照 A (07-10): 播放量 = 10,000,000
快照 B (07-16): 播放量 = 11,352,608
→ 07-10 ~ 07-16 增量 = 1,352,608 (日均 +193,230)

逻辑: 每次上传星图数据保存为带时间戳的快照，通过两快照相减得到区间增量。
```

### 9.3 混合页面时间对齐

以"原生内容加热"页面为例：

```
声量宝(巨量引擎-按日):   07-01~07-16 → 直接查询 ✅
星智投(星图-累计快照):   截止07-16快照 → 按发布时间筛选内容 + 最新快照 ⚠️
DOU+(手工填报):          07-01~07-16 → 按投放日期筛选 ✅

→ 页面顶部显示统一时间范围，各产品卡片标注各自的数据口径
```

### 9.4 时间模式标识

```
📊 按日聚合 (07-01 ~ 07-16)     ← 千川/巨量引擎/罗盘
📸 累计快照 (截止 07-16)        ← 星图
🔢 增量推算 (07-10 → 07-16)     ← 星图两次快照差值
📋 快照周 (2026-W28)            ← 云图
✋ 手工填报                      ← DOU+
```
